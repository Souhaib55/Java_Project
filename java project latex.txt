\chapter{Introduction Générale}

\section{Contexte du projet}

Dans le cadre du module de \textbf{Programmation Java}, il nous est demandé de concevoir et développer une application mettant en œuvre les concepts fondamentaux de la programmation orientée objet, l'utilisation des collections Java, ainsi que des techniques avancées telles que la persistance de données et les interfaces graphiques.

Le projet choisi est un \textbf{Système de Quiz Interactif} : une application complète permettant de créer, gérer et passer des quiz à choix multiples, avec suivi des scores et classement des utilisateurs.

\section{Problématique}

Les systèmes d'évaluation traditionnels manquent souvent d'interactivité et de flexibilité. Comment concevoir une application Java qui permette à la fois :
\begin{itemize}[noitemsep]
    \item La création et la gestion dynamique de questions par un administrateur ?
    \item La passation interactive de quiz par des utilisateurs ?
    \item Le suivi et la persistance des scores dans le temps ?
    \item Une expérience utilisateur agréable via une interface graphique ?
\end{itemize}

\section{Objectifs du projet}

\subsection{Objectifs principaux}
\begin{enumerate}[noitemsep]
    \item Développer un système complet de quiz interactif en Java.
    \item Mettre en œuvre les concepts de POO : héritage, abstraction, polymorphisme, encapsulation.
    \item Utiliser les collections Java (\texttt{List}, \texttt{Map}, \texttt{Set}) avec généricité.
    \item Implémenter la persistance des données via fichiers CSV et base de données MySQL (JDBC).
    \item Concevoir une interface graphique avec JavaFX.
\end{enumerate}

\subsection{Objectifs secondaires}
\begin{enumerate}[noitemsep]
    \item Intégrer un système de chronomètre par question.
    \item Proposer un classement (leaderboard) des meilleurs scores.
    \item Permettre la catégorisation et le filtrage des questions par thème et difficulté.
    \item Exporter les résultats en fichier texte.
\end{enumerate}

\hline
\textbf{Acteur principal} & Joueur \\
\hline
\textbf{Préconditions} & Le joueur est connecté. Il existe des questions dans la banque. \\
\hline
\textbf{Scénario principal} &
\begin{enumerate}[noitemsep, topsep=0pt]
    \item Le joueur sélectionne une catégorie.
    \item Le joueur choisit un niveau de difficulté.
    \item Le système génère un quiz de 10 questions aléatoires.

    \item Après la dernière question, le système calcule et affiche le score.
    \item Le score est sauvegardé dans l'historique.
\end{enumerate} \\
\hline
\textbf{Scénario alternatif} & Si le chronomètre expire, la réponse est considérée comme incorrecte. \\

\chapter{Contraintes et Gestion des Risques}

\section{Contraintes techniques}

\begin{itemize}[noitemsep]
    \item Le projet doit être développé en \textbf{Java 17} ou supérieur.
    \item L'utilisation des collections Java (\texttt{List}, \texttt{Map}) est \textbf{obligatoire}.
    \item La persistance doit être assurée par au moins un mécanisme (fichier ou base de données).
    \item L'application doit pouvoir fonctionner sans connexion Internet (base locale MySQL).
\end{itemize}

\section{Contraintes organisationnelles}

\begin{itemize}[noitemsep]
    \item Projet par groupe.
    \item Délai de réalisation limité (environ un mois).
    \item Livraison du code source + démonstration.
\end{itemize}

\chapter{Conclusion}

Ce cahier des charges définit le cadre complet du projet \textbf{Système de Quiz Interactif}. Le projet permettra de mettre en pratique l'ensemble des compétences acquises en programmation Java :

\begin{itemize}[noitemsep]
    \item \textbf{Programmation orientée objet} : classes abstraites, interfaces, héritage, polymorphisme.
    \item \textbf{Collections Java} : \texttt{List<Question>}, \texttt{Map<String, List<ScoreRecord>>} avec généricité.
    \item \textbf{Persistance} : fichiers CSV et base de données MySQL via JDBC.
    \item \textbf{Interface graphique} : application JavaFX multi-écrans.
    \item \textbf{Architecture logicielle} : séparation en couches (modèle, DAO, service, UI).
\end{itemize}

L'approche incrémentale adoptée garantit qu'un livrable fonctionnel sera disponible à chaque phase du développement, minimisant ainsi les risques liés aux délais.
